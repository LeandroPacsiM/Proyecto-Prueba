## What I Do

I create **production-ready README.md files** that:
- Clearly explain what the project does
- Show how to use it
- Document features and technologies
- Provide setup instructions
- Include visual elements (badges, screenshots, diagrams)
- Follow best practices for open source documentation

## When to Use Me

Use this skill when:
- ✅ A project is complete and ready for GitHub
- ✅ You need to document a new repository
- ✅ The README is missing or outdated
- ✅ You want a professional, polished documentation

## Content Guidelines

### Title
- Clear and descriptive
- Match the repository name
- Include emoji if appropriate

### Tagline
- One sentence maximum
- Explain the value proposition
- Use `>` blockquote format

### Badges
Use shields.io for:
- Status (complete, in-development, deprecated)
- License (MIT, Apache, GPL)
- Version
- Build status (if applicable)

### About Section
Answer these questions:
1. What does this project do?
2. What problem does it solve?
3. Who is it for?

### Features
- Use bullet points
- Start with strong verbs
- Highlight unique aspects
- Group related features

### Technologies
- Use a table for clarity
- Categorize (Frontend, Backend, Tools, etc.)
- Include version numbers if relevant

### Project Structure
- Show the directory tree
- Explain key files
- Keep it concise

### Getting Started
- Be specific about prerequisites
- Step-by-step installation
- How to run/test/build

### Tone
- Professional but approachable
- Active voice
- Present tense
- Inclusive language

## Examples

### Good Feature Description
```markdown
- **Interactive Terminal**: Built-in command-line interface with 
  10+ commands (help, about, projects, contact) that respond 
  instantly with styled output
```

### Bad Feature Description
```markdown
- Has a terminal
```

## Variables to Replace

Always replace these placeholders:
- `Project Title` → Actual project name
- `yourusername` → GitHub username
- `project-name` → Repository name
- `Your Name` → Author name
- `your.email@example.com` → Contact email
- `./assets/images/demo.png` → Actual screenshot path

## Special Considerations

### For Portfolio Projects
- Emphasize skills demonstrated
- Include live demo link prominently
- Mention if it's open for hire
- Add contact section

### For Matrix-Themed Projects
- Use green/black color scheme in badges
- Reference the aesthetic in description
- Mention special effects (digital rain, glitch, etc.)

### For Open Source
- Clear contribution guidelines
- Code of conduct reference
- Issue template mention

## Output Format

- UTF-8 encoding
- GitHub Flavored Markdown
- Max 80 characters per line (for readability)
- Consistent emoji usage (one per section)
- Proper heading hierarchy (H1 → H2 → H3)
